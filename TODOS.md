**Why:** Right now stubs are always blank. Once you control skills that have well-known

## P2: Configurable external diff tool for `dfiles diff`

**What:** Allow users to configure an external diff viewer (e.g. `delta`, `diff-so-fancy`) via `dfiles.toml` under a `[diff]` section: `tool = "delta"`.

**Why:** Some users have strong preferences for diff rendering. `dfiles diff` ships with built-in ANSI colors, but external tools like `delta` provide syntax highlighting, side-by-side views, and git-style decorations that can't be replicated in-process.

**Cons:** Requires piping the combined diff output through an external process, which complicates the output pipeline slightly. Also requires adding a `[diff]` config section to `DfilesConfig` (new schema fields).

**Context:** `dfiles diff` landed with built-in `--color=auto|always|never` and `--stat`. External tool support is the next logical step once users start using the feature in practice. Implementation: pipe the full formatted diff string through `Command::new(tool).stdin(Stdio::piped()).spawn()` before printing. Only applies when the tool is configured and stdout is a tty.

**Depends on / blocked by:** ~~`dfiles diff` shipping~~ â€” unblocked (`dfiles diff` shipped 2026-03-20).
