# dfiles v0.1.0 Release Notes

**Release date:** 2026-03-20 (initial release)

---

## Overview

dfiles is an AI-first developer environment manager. It tracks dotfiles,
Homebrew packages, language runtimes, and Claude Code skills in a single git
repository, and reproduces your complete development environment on any machine
from a single command.

```sh
dfiles init gh:you/my-env --apply  # new machine: full environment in minutes
dfiles apply                        # sync changes to this machine
dfiles status                       # see what's drifted
```

---

## What ships in v0.1.0

### Core dotfile management

**`dfiles init`** — create or clone a dfiles repo.

```sh
dfiles init                          # blank scaffold
dfiles init gh:you/my-env            # clone from GitHub
dfiles init gh:you/my-env --apply    # clone and apply immediately
dfiles init https://github.com/...   # any git URL
```

**`dfiles add`** — start tracking a file or directory.

```sh
dfiles add ~/.zshrc
dfiles add ~/.config/git/config
dfiles add ~/.config/nvim             # prompts: extdir or recursive
dfiles add ~/.zshrc --link --apply    # symlink, apply immediately
```

Sensitive filename detection (`.env`, `id_rsa`, `.pem`, etc.) prompts before
saving. Directory add detects git remotes and offers to track as an external
clone (`extdir_`) or recursively copy files.

**`dfiles apply`** — deploy to this machine.

```sh
dfiles apply                    # all sections
dfiles apply --dry-run          # preview without writing
dfiles apply --files            # dotfiles only
dfiles apply --brews            # Homebrew only
dfiles apply --ai               # AI skills only
dfiles apply --run-scripts      # also execute source/scripts/
```

Backs up existing files before overwriting. Private files are chmod 0600,
executables are chmod 0755, symlinks are linked to their source entry.

**`dfiles diff`** — show what's drifted.

```sh
dfiles diff             # full diff
dfiles diff --stat      # summary only
dfiles diff --files     # dotfiles only
dfiles diff --color always
```

Exits 1 when drift is found (CI-friendly).

**`dfiles status`** — concise drift view.

```
✓  ~/.zshrc
M  ~/.gitconfig   (modified)
?  ~/.config/nvim (missing)
```

### Magic-name file encoding

dfiles encodes file metadata in the source filename itself — no separate
registry. Compatible with chezmoi's encoding.

```
source/dot_zshrc                     →  ~/.zshrc
source/private_dot_ssh/id_rsa        →  ~/.ssh/id_rsa       (chmod 0600)
source/executable_dot_local/bin/foo  →  ~/.local/bin/foo     (chmod 0755)
source/symlink_vscode_settings.json  →  ~/vscode_settings.json (symlink)
source/dot_gitconfig.tmpl            →  ~/.gitconfig          (Tera template)
source/extdir_dot_config_nvim        →  ~/.config/nvim        (git clone)
```

---

### Tera templates

`.tmpl` files are rendered through the Tera engine before writing. Available
variables:

| Variable | Example |
|----------|---------|
| `hostname` | `mbp.local` |
| `username` | `alice` |
| `os` | `macos` or `linux` |
| `arch` | `aarch64` or `x86_64` |
| `source_dir` | `/Users/alice/.local/share/dfiles` |
| `profile` | `default` |
| `env("VAR")` | value of `$VAR` |
| `op(path="op://...")` | 1Password secret (see below) |

---

### 1Password integration

```
{{ op(path="op://Personal/GitHub/token") }}
```

Secrets are resolved at apply time via the `op` CLI. The source file stores
the reference, not the value — safe to commit.

### Homebrew management

```sh
dfiles brew install ripgrep          # brew install + add to Brewfile
dfiles brew install iterm2 --cask    # cask support
dfiles brew uninstall ripgrep        # brew uninstall + remove from all Brewfiles
```

`dfiles apply --brews` runs `brew bundle install` for all Brewfiles in the
active profile. `--remove-unreferenced-brews` / `--interactive` prune
unreferenced leaf packages.

---

### AI skills management

```sh
dfiles ai add gh:anthropics/skills/pdf-processing
dfiles ai add gh:owner/repo --platforms claude-code,codex
dfiles ai fetch              # download to cache without deploying
dfiles ai update             # re-fetch latest + update lock SHAs
dfiles ai remove my-skill
dfiles ai discover           # detect installed AI platforms
```

Skills are declared in `ai/skills/<name>/skill.toml` and deployed to platform
skill directories (`~/.claude/skills/`) on `dfiles apply --ai`. Every `gh:`
source is pinned by SHA in `dfiles.lock`.

### Profiles and modules

```toml
# dfiles.toml
[profile.default]
modules = ["shell", "git"]

[profile.work]
extends = "default"
modules = ["secrets"]
```

Profiles control which modules (and therefore which Brewfiles and mise configs)
are active on a given machine. The last-used profile is saved in state and
reused automatically.

---

### chezmoi import

```sh
dfiles import --from chezmoi
dfiles import --from chezmoi --dry-run
dfiles import --from chezmoi --include-ignored-files
```

Full import pipeline from chezmoi source directories:

- `dot_` / `private_` / `executable_` / `symlink_` prefixes converted
- Go template syntax converted to Tera where possible
- `.chezmoiexternal.toml` git entries imported as `extdir_` markers
- `.chezmoiignore` written as `config/ignore`
- `run_once_` scripts scanned for `brew bundle` and `mise install` patterns;
  module TOMLs auto-generated where possible
- All scripts copied to `source/scripts/` for `dfiles apply --run-scripts`

---

### Cross-platform binaries and installer

```sh
curl -fsSL https://raw.githubusercontent.com/johnstegeman/dfiles/main/install.sh | sh
```

SHA256-verified installer. Supports macOS (x86_64, arm64) and Linux (x86_64,
aarch64, i686, armv7 musl). Installs to `/usr/local/bin` or `~/.local/bin`
with automatic PATH detection.
