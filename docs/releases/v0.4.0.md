# haven v0.4.0 Release Notes

**Release date:** 2026-03-23

---

## Highlights

### The project is now called haven

The tool has been renamed from `dfiles` to `haven`. The name change is the primary
reason for the 0.4.0 version bump.

**Why?** "dfiles" sounds like "defiles" when spoken aloud, and `df` conflicts with
the standard Unix `df` (disk free) command. `haven` — a place of shelter, a home
for your configuration — is a better fit for what the tool does.

**What changed:**

| Before | After |
|--------|-------|
| `dfiles` (binary) | `haven` |
| `dfiles.toml` | `haven.toml` |
| `~/.dfiles/` | `~/.haven/` |
| `DFILES_DIR` | `HAVEN_DIR` |
| `DFILES_VCS` | `HAVEN_VCS` |
| `DFILES_TELEMETRY` | `HAVEN_TELEMETRY` |
| `DFILES_PROFILE` | `HAVEN_PROFILE` |
| GitHub `johnstegeman/dfiles` | `johnstegeman/haven` |

The source encoding, repo layout, and file format are **unchanged**. Existing repos
migrate by renaming `dfiles.toml` → `haven.toml` and `~/.dfiles/` → `~/.haven/`.

**New installer URL:**

```sh
curl -fsSL https://raw.githubusercontent.com/johnstegeman/haven/main/install.sh | sh
```

---

### `config/ignore` is now a Tera template

The ignore file at `config/ignore` is now rendered through the Tera engine before
patterns are evaluated — exactly how chezmoi handles `.chezmoiignore`. This means
you can write OS- or hostname-conditional ignore patterns:

```
{% if os == "macos" %}
.DS_Store
{% endif %}

{% if hostname == "work-laptop" %}
.config/work-secrets/**
{% endif %}
```

All variables available in `.tmpl` files (`os`, `hostname`, `username`, `profile`,
`data.*`) work in `config/ignore` too. If rendering fails, haven warns and falls
back to ignoring nothing (safe degradation, not a crash).

`haven import --from chezmoi` now converts Go template syntax in `.chezmoiignore`
to Tera syntax rather than stripping template lines.

---

### `haven init` is now a safe no-op on an existing repo

Running `haven init` in a directory that's already a haven repo used to error out.
It now prints an informational message and exits 0. This makes it safe to call
idempotently in bootstrap scripts.

---

### `haven --version` shows full build identity

```
haven 0.4.0+abc1234
```

The version output now includes the short git commit hash, matching the `version`
field in telemetry events. Useful when filing bug reports.

---

## Upgrade

```sh
# Re-install from the new URL
curl -fsSL https://raw.githubusercontent.com/johnstegeman/haven/main/install.sh | sh

# Rename your config and state directory
mv ~/haven.toml ~/.local/share/haven/haven.toml 2>/dev/null || true
mv ~/.dfiles ~/.haven 2>/dev/null || true
```
