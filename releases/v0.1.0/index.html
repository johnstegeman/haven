haven v0.1.0 Release Notes¶

Release date: 2026-03-20 (initial release)

Overview¶

haven is an AI-first developer environment manager. It tracks dotfiles, Homebrew packages, language runtimes, and Claude Code skills in a single git repository, and reproduces your complete development environment on any machine from a single command.

haven init gh:you/my-env --apply  # new machine: full environment in minutes
haven apply                        # sync changes to this machine
haven status                       # see what's drifted

What ships in v0.1.0¶

Core dotfile management¶

haven init — create or clone a haven repo.

haven init                          # blank scaffold
haven init gh:you/my-env            # clone from GitHub
haven init gh:you/my-env --apply    # clone and apply immediately
haven init https://github.com/...   # any git URL

haven add — start tracking a file or directory.

haven add ~/.zshrc
haven add ~/.config/git/config
haven add ~/.config/nvim             # prompts: extdir or recursive
haven add ~/.zshrc --link --apply    # symlink, apply immediately

Sensitive filename detection (.env, id_rsa, .pem, etc.) prompts before saving. Directory add detects git remotes and offers to track as an external clone (extdir_) or recursively copy files.

haven apply — deploy to this machine.

haven apply                    # all sections
haven apply --dry-run          # preview without writing
haven apply --files            # dotfiles only
haven apply --brews            # Homebrew only
haven apply --ai               # AI skills only
haven apply --run-scripts      # also execute source/scripts/

Backs up existing files before overwriting. Private files are chmod 0600, executables are chmod 0755, symlinks are linked to their source entry.

haven diff — show what's drifted.

haven diff             # full diff
haven diff --stat      # summary only
haven diff --files     # dotfiles only
haven diff --color always

Exits 1 when drift is found (CI-friendly).

haven status — concise drift view.

✓  ~/.zshrc
M  ~/.gitconfig   (modified)
?  ~/.config/nvim (missing)

Magic-name file encoding¶

haven encodes file metadata in the source filename itself — no separate registry. Compatible with chezmoi's encoding.

source/dot_zshrc                     →  ~/.zshrc
source/private_dot_ssh/id_rsa        →  ~/.ssh/id_rsa       (chmod 0600)
source/executable_dot_local/bin/foo  →  ~/.local/bin/foo     (chmod 0755)
source/symlink_vscode_settings.json  →  ~/vscode_settings.json (symlink)
source/dot_gitconfig.tmpl            →  ~/.gitconfig          (Tera template)
source/extdir_dot_config_nvim        →  ~/.config/nvim        (git clone)

Tera templates¶

.tmpl files are rendered through the Tera engine before writing. Available variables:

Variable Example
hostname mbp.local
username alice
os macos or linux
arch aarch64 or x86_64
source_dir /Users/alice/.local/share/haven
profile default
env("VAR") value of $VAR
op(path="op://...") 1Password secret (see below)

1Password integration¶

{{ op(path="op://Personal/GitHub/token") }}

Secrets are resolved at apply time via the op CLI. The source file stores the reference, not the value — safe to commit.

Homebrew management¶

haven brew install ripgrep          # brew install + add to Brewfile
haven brew install iterm2 --cask    # cask support
haven brew uninstall ripgrep        # brew uninstall + remove from all Brewfiles

haven apply --brews runs brew bundle install for all Brewfiles in the active profile. --remove-unreferenced-brews / --interactive prune unreferenced leaf packages.

AI skills management¶

haven ai add gh:anthropics/skills/pdf-processing
haven ai add gh:owner/repo --platforms claude-code,codex
haven ai fetch              # download to cache without deploying
haven ai update             # re-fetch latest + update lock SHAs
haven ai remove my-skill
haven ai discover           # detect installed AI platforms

Skills are declared in ai/skills/<name>/skill.toml and deployed to platform skill directories (~/.claude/skills/) on haven apply --ai. Every gh: source is pinned by SHA in haven.lock.

Profiles and modules¶

# haven.toml
[profile.default]
modules = ["shell", "git"]

[profile.work]
extends = "default"
modules = ["secrets"]

Profiles control which modules (and therefore which Brewfiles and mise configs) are active on a given machine. The last-used profile is saved in state and reused automatically.

chezmoi import¶

haven import --from chezmoi
haven import --from chezmoi --dry-run
haven import --from chezmoi --include-ignored-files

Full import pipeline from chezmoi source directories:

  • dot_ / private_ / executable_ / symlink_ prefixes converted
  • Go template syntax converted to Tera where possible
  • .chezmoiexternal.toml git entries imported as extdir_ markers
  • .chezmoiignore written as config/ignore
  • run_once_ scripts scanned for brew bundle and mise install patterns; module TOMLs auto-generated where possible
  • All scripts copied to source/scripts/ for haven apply --run-scripts

Cross-platform binaries and installer¶

curl -fsSL https://raw.githubusercontent.com/johnstegeman/haven/main/install.sh | sh

SHA256-verified installer. Supports macOS (x86_64, arm64) and Linux (x86_64, aarch64, i686, armv7 musl). Installs to /usr/local/bin or ~/.local/bin with automatic PATH detection.
